Modelo de Datos

Estructura de la Base de Datos

La base de datos está organizada en múltiples schemas para garantizar el aislamiento de datos entre edificios:

Schema Público (public)

Users

Gestión de usuarios y autenticación

Buildings

Registro de edificios y referencias a sus schemas

Schemas de Edificios

building_abc123
- Units (Unidades)
- Claims (Reclamos)
building_xyz789
- Units (Unidades)
- Claims (Reclamos)

Schema Público (public)

// Usuarios y autenticación model User { id UUID @id email String @unique password String firstName String lastName String role UserRole createdAt DateTime updatedAt DateTime }
// Edificios model Building { id UUID @id name String address String schema String @unique adminId UUID status BuildingStatus createdAt DateTime updatedAt DateTime }

Schema de Edificio (building_*)

Cada edificio tiene su propio schema con estas tablas:

// Unidades/departamentos model Unit { id UUID @id number String floor String ownerId UUID createdAt DateTime updatedAt DateTime owner User @relation(fields: [ownerId], references: [id]) }
// Reclamos/tickets model Claim { id UUID @id title String description String status String unitId UUID creatorId UUID comments String[] images String[] createdAt DateTime updatedAt DateTime }
